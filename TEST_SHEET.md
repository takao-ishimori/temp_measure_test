## 2. テスト項目一覧

### 2.1 単体テスト：DHT22 センサー読み取り

| No | テスト項目 | テスト内容 | 合格基準 | 結果 | 備考 |
|----|-----------|-----------|---------|------|------|
| T01 | センサー接続確認 | GPIO4 の信号が検出できるか | エラーなく初期化できる | [x] 合格 [ ] 不合格 | 初回から正常読み取りに成功 |
| T02 | 温度読み取り | 1秒間隔で10回温度を読み取る | 全10回の値が 0〜50℃ の範囲内 | [x] 合格 [ ] 不合格 | 29.1〜29.8℃ 範囲内 |
| T03 | 湿度読み取り | 1秒間隔で10回湿度を読み取る | 全10回の値が 0〜100% の範囲内 | [x] 合格 [ ] 不合格 | 53.4〜55.3% 範囲内 |
| T04 | 連続動作安定性 | 5分間連続で読み取りを続ける | エラー・異常停止が発生しない | [ ] 合格 [ ] 不合格 | 未実施（次回） |
| T05 | 読み取りエラー耐性 | DHT22は約2秒間隔以上でないとエラーが出る仕様。1.5秒間隔で30回試行 | 50%以上読み取り成功 | [ ] 合格 [ ] 不合格 | 未実施（次回） |
| T06 | 異常値検出 | センサーを指で温めた際に温度が上昇するか | 温度が明らかに上昇する（目視確認） | [x] 合格 [ ] 不合格 | 温度 29.1→29.3℃ 上昇。湿度も 54.7→62.5% へ敏感に反応。接触時に通信エラー増加（センサー特性として正常） |
| T07 | Python/C++結果一致 | 同一環境でPython版とC++版の読み取り値を比較 | 差が ±0.5℃ 以内 | [x] 合格 [ ] 不合格 | 温度差 0.3℃（時間差による室温変動の可能性あり）。湿度差 0.1%以内。実質一致と判断 |

## 3. テスト実行手順

### 3.1 Python版

```bash
cd /path/to/rasbperrypi
python sensor/dht22.py
```

### 3.2 C++版

```bash
cd /path/to/rasbperrypi
g++ -std=c++11 -o sensor/dht22 sensor/dht22.cpp -lwiringPi
sudo ./sensor/dht22
```

### 3.3 モック実行（ハードウェアなし検証用）

```bash
python sensor/dht22_mock.py
```
