# 面接カンペ - temp_measure

## 30秒自己紹介

「IoT温度監視システムを自作しました。Raspberry PiでDHT22センサーを制御し、
MQTT通信→SQLite保存→Webダッシュボード→Zabbix監視まで一貫実装しています。
Python/C++両方で書いてます。GitHubで公開中です。」

---

## システムの流れ（一言で言うなら）

「センサーが取った温度をMQTTという軽量通信で飛ばして、
データベースにためて、ブラウザのグラフで見られるようにしました。
さらにZabbixという監視ツールでラズパイ自体のCPUやメモリも監視しています。」

---

## 各コンポーネント（質問されたら）

| 聞かれたら | こう答える |
|-----------|-----------|
| DHT22って？ | 1-Wire通信の温湿度センサーです。GPIO一本で双方向通信します |
| なぜC++も？ | Pythonよりタイミング制御が正確で、読み取り失敗が減ります |
| MQTTって？ | IoTで標準の軽量メッセージプロトコルです。publish/subscribe型 |
| Mosquittoは？ | オープンソースのMQTTブローカーです。自分でラズパイに立てました |
| SQLiteの理由は？ | 小規模データなら軽量で十分。外部DB不要でラズパイ1台で完結します |
| Chart.jsは？ | JavaScriptのグラフライブラリ。ブラウザでリアルタイム更新できます |
| Zabbixって？ | サーバー監視の定番ツール。CPU/メモリ/ディスクを自動収集します |
| t検定とは？ | 前半と後半のデータで温度に有意差があるか統計的に判定しました |

---

## ファイル構成の覚え方（上から下へ）

```
sensor/     → センサーからデータを取る（入口）
mqtt/       → データを届ける（通信）
database/   → データをためる（保存）
monitoring/ → データを見せる＋監視する（出口）
analytics/  → データを分析する（統計）
opencv/     → アナログ温度計も読みたい（実験）
start_all.* → 全部まとめて起動（運用）
```
